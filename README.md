# Curt 减脂计算器 - 微信小程序

一款功能完整的微信小程序，专为健身爱好者设计，提供精准的热量计算、营养素分配和减脂周期预测功能。

## 项目特性

✨ **核心功能**
- 🔥 **热量计算**: 基于Mifflin-St Jeor公式精准计算基础代谢率(BMR)和每日热量消耗(TDEE)
- 📊 **营养素分配**: 支持532、442比例和自定义营养素分配方案
- 📈 **减脂预测**: 根据目标体重和减重速度预测减脂周期
- 👤 **身体数据**: 实时显示BMI指数和身体质量评估

🎨 **设计特点**
- 绿色主题配色(#2DC78F)，清爽专业
- 圆角卡片设计，视觉层次清晰
- 顶部Tab导航，交互流畅
- 响应式布局，适配各种屏幕尺寸

⚡ **技术特点**
- 原生微信小程序开发(WXML + WXSS + JS)
- 前端计算，无需后端支持
- 全局数据管理，页面间数据同步
- 本地存储，保留用户数据

## 使用说明

### 微信开发者工具预览步骤

1. **下载微信开发者工具**
   - 访问 [微信开发者工具官网](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
   - 下载并安装最新版本

2. **导入项目**
   - 打开微信开发者工具
   - 点击"导入项目"
   - 选择本项目文件夹路径
   - 输入AppID（可使用测试号）
   - 点击"导入"

3. **预览项目**
   - 项目导入后，点击顶部"预览"按钮
   - 用微信扫描生成的二维码
   - 即可在微信中预览小程序

4. **调试**
   - 使用"调试器"查看控制台输出
   - 使用"Network"查看网络请求
   - 使用"Storage"查看本地存储数据

### 功能使用流程

#### 第一步：热量计算
1. 进入"热量计算"页面
2. 输入个人信息：
   - 选择性别（男/女）
   - 输入年龄（15-100岁）
   - 输入身高（100-250cm）
   - 输入体重（30-200kg）
3. 系统自动计算基础代谢率(BMR)
4. 输入每日训练时间和强度等级
5. 选择营养素比例（532/442/自定义）
6. 查看每日热量消耗和营养素分配建议

#### 第二步：减脂预测
1. 进入"减脂预测"页面
2. 设置初始体重和目标体重
3. 调整每月减重比例（1%-5%）
4. 查看预计减脂周期和每月/每周减重计划
5. 点击"保存目标"保留数据

#### 第三步：个人信息
1. 进入"个人"页面
2. 查看每日热量建议和身体数据
3. 查看BMI指数和身体质量评估
4. 查看营养素分配建议
5. 查看减重进度

## 计算公式

### 基础代谢率(BMR) - Mifflin-St Jeor公式
- **男性**: BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 + 5
- **女性**: BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 161

### 每日热量消耗(TDEE)
- TDEE = BMR + 训练热量
- 训练热量 = 训练时间(分钟) × 强度系数(5/8/10)

### 营养素分配
- **碳水克数** = (总热量 × 碳水%) ÷ 4
- **蛋白质克数** = (总热量 × 蛋白质%) ÷ 4
- **脂肪克数** = (总热量 × 脂肪%) ÷ 9

### 减脂预测
- **总减重** = 初始体重 - 目标体重
- **预计月数** = 总减重 ÷ (初始体重 × 每月减重比例)
- **每月减重** = 总减重 ÷ 预计月数
- **每周减重** = 每月减重 ÷ 4

## 数据存储

小程序使用以下方式存储数据：

1. **全局数据** (App.globalData)
   - 实时存储用户输入的身体数据
   - 页面间共享数据

2. **本地存储** (wx.setStorageSync)
   - 保存减脂目标（目标体重、初始体重、每月减重比例）
   - 数据持久化，应用重启后仍可访问

## 后续迭代指令

### 添加新功能的步骤

1. **在app.js中添加全局数据**
   ```javascript
   globalData: {
     // 新增字段
     newField: defaultValue
   }
   ```

2. **创建新页面**
   ```
   pages/newpage/
   ├── newpage.wxml
   ├── newpage.js
   └── newpage.wxss
   ```

3. **在app.json中注册页面**
   ```json
   "pages": [
     "pages/newpage/newpage"
   ]
   ```

4. **编写页面逻辑**
   - 在JS中使用 `getApp()` 获取全局数据
   - 使用 `setData()` 更新页面数据
   - 使用 `wx.setStorageSync()` 保存数据

### 常见修改

#### 修改主题颜色
- 编辑 `app.wxss` 中的 `--primary-color: #2DC78F`
- 所有组件会自动应用新颜色

#### 添加新的计算公式
- 在 `app.js` 中添加新的计算方法
- 在页面中调用该方法
- 使用 `setData()` 更新显示结果

#### 修改导航栏
- 编辑 `app.json` 中的 `tabBar` 配置
- 添加或删除标签页
- 更新页面路径和图标

## 常见问题

### Q: 如何修改小程序名称？
A: 编辑 `app.json` 中的 `navigationBarTitleText` 字段

### Q: 如何添加新的性别选项？
A: 在 `app.js` 中修改 `gender` 字段的可选值，然后在页面中添加对应的UI

### Q: 如何修改营养素比例？
A: 在 `app.js` 的 `getNutritionBreakdown()` 方法中修改比例数值

### Q: 如何保存用户数据到服务器？
A: 需要升级为有后端的小程序，使用 `wx.request()` 调用API
